The SRoCS Platform

  • SRoCS: Swarm Robotics Construction System
  • Mobile robots that assemble stigmergic blocks into structures
  • Stigmergic blocks are markable by the robots

Challenges to Moving Forward

  • The differential drive system was unreliable
  • Custom built hardware is expensive
  • Running experiments is time-consuming

Potential Solutions

  1. Improve the hardware
    • Hardware-based experiments are more convincing
    • Recent technology may reduce manufacturing costs
    • Development remains time-consuming and expensive
  2. Perform experiments in simulation
    • Faster development
    • Potential simulation-to-reality gap

The ARGoS Simulator

  • Highly modular and extendable
  • Robot controllers written in C++ or Lua
  • Supports multiple physics engines and visualizations
  • In contrast to USARSim, Gazebo, and Webots, ARGoS can simulate 10,000 robots in faster than real time

Extensions to the ARGoS Simulator

  • The Dynamics 3D Plugin
  • The Prototyping Plugin
  • The Radio Plugin
  • The Camera Plugin

The 3D Dynamics Plugin

The Prototyping Plugin

  • Robots instantiated using just XML
  • Robots can include cameras, proximity sensors, radios, LEDs, and tags
  • Provides models for the Dynamics 3D plugin and Qt-OpenGL visualization
  • Joints can be measured and controlled via C++ and Lua

The Radio Plugin

  • Simple point-to-point radio
  • Send multiple messages of arbitrary length
  • Transmission can be seen from the visualization
  • Supported in both C++ and Lua robot controllers

The Camera Plugin

  • Supports any existing robot in ARGoS
  • Multiple cameras
  • Detection algorithms for tags and LEDs
  • Adding algorithms is easy (often only a few lines of code)
  • The frustum of the camera can be seen from visualization

Simulation of the SRoCS Platform

  • Stigmergic block
    • Uses prototyping plugin
    • Radio plugin for NFC communication
    • Contains tags and LEDs
  • Mobile Robot
    • Uses the prototyping plugin
    • Detects the tags and LEDs on a block via camera plugin
    • Communication with a block using the radio plugin

The Staircase Construction Task

  • Same state machine as the hardware
  • Qualitative replication of the hardware-based experiment

The Pyramid Construction Task

  • Extension of the state machine from staircase task
  • Enables experiments that were not possible with the hardware

Conclusion

  • Summary
    • The developed extensions facilitate a qualitative simulation of the SRoCS platform
    • Extensions are generic and may be used to simulate other systems
  • Future Work
    • Improve the hardware so that there is a reliable system that can be modelled in simulation
    • Investigate the simulation-to-reality gap
